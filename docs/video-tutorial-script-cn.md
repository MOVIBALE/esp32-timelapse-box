# 新手视频教程脚本

ESP32 延时摄影盒子完整录制版。

目标：用一条视频说明 S3 Sony BLE 和兼容型 C3 两条路线。主线始终是
“连接 -> 安全配置 -> dry-run -> 真实层事件 -> armed 或恢复”，不要把
raw REPL 或串口刷屏作为新手主画面。

## 开场：它解决什么

画面：打印机、ESP32 盒子、相机、配置器工作台。

口播：

> ESP32 延时摄影盒子在局域网里读取 Klipper 的层事件，再触发相机。
> 打印时不需要电脑常驻监听。我们先做 dry-run，证明层事件正确后才允许
> 正式拍摄。

字幕：`v0.1.0 工程预发布；先 dry-run，后 armed。`

## 1. 准备 Klipper

准备材料：ESP32-S3、USB 数据线、Sony 相机和可访问 Moonraker 的 Klipper
打印机。先打开 `README-FIRST.md`，确认使用的是当前发布包。

画面：展示 `config/klipper/esp32_timelapse.cfg` 和 `printer.cfg` include。

口播：

> 先安装统一宏 `ESP_TIMELAPSE_SHOT`。旧设备的兼容宏还保留，但新切片
> 都应该调用这个新名称。

确认 Moonraker 能读取宏对象，暂时不要手动触发快门。

## 2. 启动浏览器配置器

画面：根目录 `START-WINDOWS.cmd`、`START-MAC.command`，然后进入页面。

口播：

> Windows 双击 START-WINDOWS，macOS 运行 START-MAC。页面只在本机
> localhost 打开。必须用 Chrome 或 Edge，因为我们直接使用 Web Serial。

说明：启动器不会自动打开串口、刷固件或拍照。
页面里的“教程检查清单”会标出当前步骤，新手按顺序完成即可。

## 3. 选择硬件路线

### S3 路线

画面：选择 `ESP32-S3 + Sony BLE`。

材料：ESP32-S3、USB 数据线、已开启蓝牙遥控的 Sony 相机。

口播：

> S3 是主动连接 Sony 相机的主路线。固件是 ESP-IDF，网络、配对、重连和
> dry-run 都通过浏览器串口命令完成。

### C3 路线

画面：切到 `兼容型 ESP32-C3 快门盒`。

口播：

> C3 路线服务已有 MicroPython 快门盒，保留原来的 GPIO 或 HID 快门输出。
> 它可以选 HTTP 轮询或 WebSocket 代理，但不会把普通空白 C3 自动变成
> Sony 蓝牙遥控器。

演示完区别后，断开串口，再选本期实际使用的路线。

## 4A. S3 首次配置与配对

1. 点击“连接板子”，在 Chrome 弹窗中选择 ESP32 串口；
2. 指出页面连接后只读取状态，不会擅自改变 dry-run/armed；
3. 填 Wi-Fi 和 Moonraker，点击“配置网络并保持 dry-run”；
4. 在 Sony 上打开蓝牙遥控配对界面；
5. 点击“首次配对 Sony”；
6. 相机出现提示后，在相机上确认；
7. 等日志显示“首次配对完成”，再等 `ready=true`。

口播：

> 首次配对按钮只做绑定，不写快门通道，所以不会拍照。以后开机不要重复
> 配对，打开相机后等待盒子自动重连；连接按钮只用于重试。

若没有相机提示：保持配对界面，等待上一轮扫描结束后重试。不要用“连接已
配对 Sony”代替首次配对。

## 4B. C3 安全上传

1. 连接串口；
2. 填 Wi-Fi、密码和 Moonraker；
3. 先选择 HTTP 轮询；
4. 点击“上传 disabled / dry-run”；
5. 软重启，确认 `enabled=false, dry_run=true`；
6. 点击“启用 dry-run 监听”并再次软重启。

口播：

> 配置器会自动处理 raw REPL，新手不需要打开高级调试。恢复时只删除
> main.py，不碰 boot.py。

## 5. 观察真实层变化

画面：切一个 10 到 20 层小模型，开始真实打印。工作台显示层号和事件数。

口播：

> 现在虽然是真打印，但盒子仍是 dry-run。我们至少等一个真实层事件，
> 更稳妥可以看三到十层。S3 相机和 C3 快门此时都不应该动作。
> 没确认 DRY_RUN 日志前不要 armed，dry-run 不会触发快门。

确认：

- 宏源优先显示 `ESP_TIMELAPSE_SHOT`；
- 日志显示 `dry_run=true` 或 `DRY_RUN`；
- 层号连续；
- 相机没有新增照片；
- 没有重启或重复事件。

## 6. Traditional 与 Smooth

画面：切片器中的关闭、传统、平滑三种模式及 G-code 预览。

口播：

> 传统模式在完成层后原地等运动结束再拍。平滑模式会生成真实稳定塔，完成
> 当前层和塔后抬 Z、停车拍照，再安全返回。单材料平滑模式也要有稳定塔。

字幕：`最终 purge 在最后一帧之前；每个完成层一帧。`

## 7. 正式触发闸门

画面：Sony 为 `ready=true`，dry-run 事件数大于 0，输入固定短语。

口播：

> 只有 dry-run、相机画面、存储卡、电量都确认后，才输入
> `ARM DRY-RUN VERIFIED`。进入 armed 后，下一次有效层事件会真实拍照。

实拍结束立刻点击“锁定为 dry-run”。调整相机、移动打印机或排查问题前也
必须先锁回 dry-run。

## 8. 常见问题、诊断与恢复官方状态

画面：复制诊断报告，展示密码被替换为星号。

口播：

> 遇到问题点“复制报告”，不要追着滚动日志手抄。报告会保留状态和事件，
> 隐藏 Wi-Fi 密码与设备地址。

C3 不再使用时可“删除 main.py 恢复”；看到 `main.py 已删除` 即表示恢复完成，
不要修改 boot.py。S3 不拍摄时保持 dry-run 或断电。

高级调试不用打开。raw REPL 和临时文件探针只在普通流程无法恢复时使用。

## 收尾检查

- 画面里没有 Wi-Fi 密码、局域网 IP、串口设备地址或个人路径；
- 没有把首次配对说成每次必做；
- 没有在 dry-run 章节触发快门；
- 没有把 C3 路线说成 Sony 主动 BLE；
- 明确说明 v0.1.0 是工程预发布。
